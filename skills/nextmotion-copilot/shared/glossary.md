# Glossaire anti-jargon — Nextmotion Copilot

> Ce glossaire est utilisé par tous les modules. Règle : zéro jargon technique en sortie vers le client. Pour chaque terme, utiliser la colonne « En clair » pour reformuler la question, et le « Défaut sûr » si le client répond « je sais pas ».

| Terme | En clair (à dire au client) | Défaut sûr si "je sais pas" |
|-------|-----------------------------|-----------------------------|
| **RPPS** | Votre numéro d'identification de médecin (11 chiffres, inscrit sur votre carte CPS ou sur Ameli Pro). | Laisser vide pour l'instant, à compléter plus tard. Ne pas bloquer la configuration. |
| **Numéro d'Ordre** | Votre numéro d'inscription au Conseil de l'Ordre des Médecins (6 à 8 chiffres). Obligatoire sur les ordonnances. | Laisser vide pour l'instant. Claude le signalera si une section en a besoin. |
| **RCP** | Responsabilité Civile Professionnelle — votre assurance pro. Nécessaire dans certains documents officiels. | Laisser vide, à compléter avec votre assureur. |
| **ADELI** | Ancien numéro d'identification ADELI (9 chiffres). Remplacé par le RPPS mais encore utilisé dans certaines applications. | Laisser vide si vous avez votre numéro RPPS. |
| **slot multiplier** | Combien de patients peuvent prendre rendez-vous en même temps sur le même créneau horaire. Ex. : si vous faites des injections en groupe, vous pouvez mettre 2 ou 3. | Défaut = 1 (un patient à la fois). C'est la valeur recommandée. |
| **scope** (global / clinic_only / provider_only) | À qui s'applique ce réglage : à toute la clinique, ou seulement à un praticien ? | Défaut = `global` (s'applique à toute la clinique). À personnaliser seulement en multi-praticiens. |
| **provider_sign** | La signature du médecin qui suit le patient (celui qui rédige l'ordonnance ou le document). | Défaut = le praticien principal de la clinique. |
| **emitter_sign** | La signature de la personne qui émet le document (peut être différente du médecin traitant, ex. un directeur médical). | Défaut = identique à `provider_sign`. |
| **DKIM / SPF** | Des réglages techniques dans votre nom de domaine email pour que vos mails ne partent pas en spam. Il faut accès à la gestion DNS de votre nom de domaine. | Si vous ne gérez pas vos DNS vous-même, contacter votre hébergeur web. En attendant, utiliser l'adresse email Nextmotion par défaut. |
| **WYSIWYG** | « Ce que vous voyez est ce que vous obtenez » — un éditeur de texte visuel comme Word, où vous formattez en cliquant sur des boutons (gras, alignement, etc.) sans écrire de code. | Pas de défaut — c'est juste le nom de l'outil d'édition que vous allez utiliser. |
| **embed code** | Un bout de code HTML à coller sur votre site web pour afficher, par exemple, le bouton de réservation Nextmotion. | Si vous n'avez pas de site web, ignorer cette étape. Si vous en avez un, partager le code à votre webmaster. |
| **GTag / pixel** (Google Tag / Meta Pixel) | Un petit bout de code qu'on met sur le site de réservation pour mesurer combien de patients prennent RDV grâce à vos publicités. | Ne pas configurer si vous n'avez pas de campagnes pub actives. |
| **Fitzpatrick** | L'échelle de Fitzpatrick classe la peau de 1 (très claire) à 6 (très foncée). Elle est utilisée pour adapter certains traitements laser. | Défaut = afficher les types I à VI. Si vous faites de la médecine esthétique générale, tout garder. |
| **concaténation SMS** | Les SMS sont limités à 160 caractères. Un message plus long est automatiquement découpé et envoyé en plusieurs SMS (= concaténation). Chaque SMS supplémentaire consomme des crédits. | Défaut = activé (les messages longs sont envoyés en entier, en plusieurs parties). |
| **are_slots_unique** | Est-ce qu'un créneau horaire ne peut être réservé que par un seul type de consultation ? Ex. : si vous cochez « unique », un créneau réservé pour une injection n'est plus disponible pour une 1ère consultation en même temps. | Défaut = `false` (non unique) — un même créneau peut accepter plusieurs types de consultation. À mettre à `true` si vous travaillez avec des salles dédiées. |
| **requires_available_doctor_and_room** | La réservation en ligne vérifie qu'un praticien ET une salle sont libres en même temps pour valider le créneau. | Défaut = `true` (recommandé) — évite les conflits de réservation. |
| **treatment_package** / **protocole** | Un ensemble de plusieurs traitements regroupés avec un tarif global (ex. "Pack rajeunissement 3 séances"). | Sans protocoles, chaque traitement est vendu séparément. Configurer seulement si vous proposez des forfaits. |
| **survey_form** / **BoltNote** | Un formulaire de consultation pré-rempli, comme un questionnaire que Claude remplit à chaque rendez-vous pour suivre l'évolution du patient. | Nextmotion fournit des BoltNotes de base. Possible de personnaliser. |
| **document_template** / **modèle de document** | Le gabarit HTML/CSS de vos devis, factures, acomptes ou avoirs. Il contient des variables comme `{{patient_name}}` ou `{{quote_number}}` qui se remplissent automatiquement. | Utiliser les modèles du pack FR inclus dans ce skill. |
| **autocomplete variable** | Un champ dans un modèle de document qui se remplit automatiquement (ex. `quote_number--autocomplete` = le numéro de devis s'incrémente seul). | Ne jamais supprimer ces variables — elles garantissent la numérotation légale séquentielle. |
| **accounting_distribution** / **répartition honoraires** | Comment les recettes d'une consultation sont partagées entre les praticiens d'une clinique multi-praticiens. | Ne configurer qu'en mode multi-praticiens. En solo, ignorer. |
| **payment_medium** | Les moyens de paiement acceptés dans Nextmotion (carte, espèces, virement, chèque…). | Ajouter au moins "Carte bancaire" et "Espèces" pour commencer. |
| **post_treatment_config** | Les consignes envoyées automatiquement au patient après son rendez-vous (ex. "Évitez le soleil 48h après l'injection"). | Utiliser les consignes du pack modèles, ou personnaliser par traitement. |
| **opening_hour** / **plage horaire** | Une plage de disponibilité dans votre agenda (ex. lundi 9h-12h et 14h-18h). Un praticien peut avoir plusieurs plages par jour. | Claude propose des horaires types, à valider ou ajuster. |
| **clinic_id** | L'identifiant unique de votre clinique dans Nextmotion. Claude le récupère automatiquement, vous n'avez jamais à le saisir. | Géré automatiquement par Claude. |
| **is_enabled** | Indique si une option/feature est activée (`true`) ou non (`false`) sur votre compte Nextmotion. | Claude vérifie automatiquement avant de proposer une action. |
| **end_time** (feature) | Date d'expiration d'une option Nextmotion. Si une date approche (moins de 30 jours), Claude vous le signalera. | Contacter le support Nextmotion pour renouveler l'option. |
| **SIRET** | Votre numéro d'identification de l'entreprise / cabinet (14 chiffres). Il figure sur votre extrait Kbis ou votre déclaration d'activité. Il apparaît sur vos devis et factures. | Laisser vide pour l'instant si vous ne l'avez pas sous la main. À compléter avant d'envoyer vos premiers documents à des patients. |
| **mentions légales** (`clinic_legal_mentions`) | Texte obligatoire sur vos devis et factures : forme juridique, SIRET, numéro TVA intracommunautaire si applicable, adresse du siège. En médecine libérale : généralement le nom du médecin, adresse, RPPS. | Défaut sûr = « [Nom du médecin] — [Adresse] — RPPS : [numéro] ». À valider avec votre comptable ou votre avocat avant utilisation officielle. |
